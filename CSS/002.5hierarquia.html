Hierarquia

Digo hierarquia quando definimos o grau de importancia/prioridade que a máquina leitora do código enxerga os elementos.

Hierarquia do documento CSS

O estilo da página sempre vai priorizar primeiro o CSS Inline (estilo definido na linha de código HTML), depois CSS Interno (estilo definido no campo <style> localizado no <head> do documento HTML) e só depois o CSS Externo (estilo definido no documento .css externo no qual o documento HTML está linkado).

CSS Inline > CSS Interno > CSS Externo

Hierarquia de Seletores

Via de regra:
01 Quanto maior o parent do Seletor, maior é na hierarquia, perdendo apenas para o último child.
02 O último child é mais forte que qualquer parent sozinho
(obs: considera-se ultimo child o id, antes dele a class e só depois a tag).
03 O seletor parent + child sucessores é mais forte que qualquer parent ou child sozinho.
04 O CSS Inline é o mais forte de todos ignorando qualquer regra.

Exemplo:

  1. CSS Inline
  2. CSS Interno
  3. CSS Externo
    1. ...tagParent + tagChild + class + id
    2. tagChild + class + id
    3. class + id
    4. id
    5. class
    6. tagChild
    7. tagParent
    8. tagGlobal
    9. tagUniversal
Obs: Quando se trata da hierarquia, vai prevalecer o último código escrito.

Regra !Important

O !important é um valor, ou seja, você deve inserir junto com os valores das propriedades, e quando inserido ele ignora qualquer regra do exemplo anterior e passa a valer este.

O important é mais usado como medida paleativa ou como teste no código, tendo em vista que o uso excessivo do important não faz tanto sentido e gera muita desordem no código
