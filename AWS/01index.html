AWS - Amazon Web Service

AWS é uma plataforma de serviços de computação em nuvem oferecida pela Amazon.com. São dezenas ou centanas de serviços completos de datacenters. Os serviços são oferecidos em várias áreas geográficas e distribuídas pelo mundo.

VPC - Virtual Private Cloud

VPC é um serviço de computação em nuvem comercial que fornece aos usuários uma nuvem privada virtual. Básicamente um Agregado de Serviços em Núvem Privada.

EC2 - Elastic Compute Cloud

EC2 é uma parte central da plataforma de cloud computing da Amazon.com, Amazon Web Services. O EC2 permite que os usuários aluguem computadores virtuais nos quais rodam suas próprias aplicações

RDS - Relational Database Service

RDS é um serviço de banco de dados relacional distribuído da Amazon Web Services. É um serviço da web executado "na nuvem" projetado para simplificar a configuração, operação e escalonamento de um banco de dados relacional para uso em aplicativos.

S3 - Amazon Simple Storage Service

S3 é um serviço oferecido pela Amazon Web Services que fornece armazenamento de objetos por meio de uma interface de serviço da web. O Amazon S3 usa a mesma infraestrutura de armazenamento escalável que a Amazon.com usa para executar sua rede de comércio eletrônico.

IAM - Identity and Access Management

IAM é uma configuração de usuário, onde você pode especificar quem ou o que pode acessar serviços e recursos na AWS, gerenciar permissões refinadas de maneira centralizada e analisar o acesso para refinar as permissões na AWS.

O que é preciso para subir uma aplicação "full" AWS?

No caso, uma aplicação Spring com um Banco de Dados Relacional (MySQL).

Passo 1:

Primeiramente é preciso que a sua aplicação esteja rodando perfeitamente em testes localhost, conectando e se comportanto de forma correta ao banco de dados.

Passo 2:

Você precisa: 01- criar uma conta na AWS
02- Em VPC, criar sua VPC (salvo engano já vem com default)
03- Criar pelo menos 3 sub-nets (aba VPC), utilizando o mesma conexão de rede da sua VPC (CIDRs IPv4)
04- Criar um Security Group (aba VPC ou EC2) (salvo engano já vem com default)
05- Criar um Elastic IP (aba VPC ou EC2) caso a sua não tenha, você precisa disso pra usar o IP Público na EC2
06- Criar uma Instance EC2 com as especificações da máquina, vincular Elastic IP e Security Group (ou criar SG novo e vincular ele na VPC também)
07- Criar um Sub-Net Group (aba RDS), e vincular todas as suas 3 sub-nets
08- Criar um Database (RDS) vinculando seu Security Group e sua Sub-Net Group
09- Configurar os IPs inbound e outbound do seu Security Group para definir quem deve acessar sua EC2 ou Database.
