Branches ou Ramificações

As branches funcionam como um caule de uma arvore, onde tem o ramo mais grosso, o principal e os demais galhos que nascem dele, os ramos

No GitHub o ramo principal chama-se Master.

Resumindo quando a gente dá o comando commit criamos um versionamento da nossa Master no nosso repositório local. E quando damos um push, enviamos o versionamento para nosso origin (repositório remoto / github).

PORÉM, commitar tudo na Master pode ser perigoso, pois se caso algo der muito errado estará tudo salvo no ramo principal, a Master e você pode estragar tudo.


Por exemplo: Suponhamos que depois de 3 meses de trabalho o HTML e JS está a todo o vapor, sendo desenvolvido perfeitamente, mas sua equipe de CSS diz para você que precisa de retroceder o versionamento pois não conseguiu concluir o projeto da forma que esperava. Mas todo seu versionamento e todas as Features estão salvas na Master, a ramificação principal. Como retroceder apenas o CSS e deixar o HTML e JS intáctos? Não dá. Por isso criaremos uma Branch para cada ponto do seu site ser desenvolvido individualmente/separadamente.
