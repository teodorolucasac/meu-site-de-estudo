Comandos

Logando/Sincronizando GitHub no Git

git config --global user.name "Seu User Name" = Insere seu nome de usuário
git config --global user.email "emaildogithub@gmail.com = Insere seu email
git config --list = confere suas config, inclusive o usuario e email

Navegar entre pastas

cd pastaNome = avança uma pasta a frente
cd .. = volta uma pasta atrás
cd C:\Program Files\Git = avança todo o caminho
dir = lista todas as pastas do local

Criar pastas e arquivos

mkdir NomePasta = cria uma pasta
touch NomeArquivo.js = cria um arquivo com formato informado
touch NomePasta/NomeArquivo2.js = cria arquivo dentro de uma pasta específica

Mover arquivos

mv NomeDoArquivo /NomeDaPasta = move um arquivo para dentro da pasta mencionada

Remover arquivos e pastas

rm NomeArquivo = remove um arquivo
rm -r NomePasta = remove uma pasta
rm -rf NomePasta .git = remove uma diretorio dentro da pasta "NomePasta"
rm -rf NomePasta = remove a pasta "NomePasta" junto com diretorio git dentro dela
rmdir NomePasta = remove apenas diretorios vazios

Salvar local e remoto 1ª vez:

git init = inicia/cria um repositório remoto na pasta atual
git branch -M main = altera a branch para main

git remote add origin https://github.com/teodorolucasac/Github-Curso.git = vincula o repositório local ao remoto
git pull origin main = desce os arquivos do remoto para local pela primeira vez (gitignore / readme)

git add . (ou git add -A) = joga TODOS os arquivos modificados/criados para area de Stage
**git add -u = da update em um arquivo que já está no Stage
**git add index.html = joga apenas o arquivo html no stage
**git restore --staged nomeDoArquivo.txt ou c:/caminhoda/pasta01 = retira um arquivo ou pasta específica do Stage

git commit -m "Nome explicativo do Commit" = cria um commit
git push -u origin main = faz o push para o repositório remoto pela primeira vez

Push e Pull pela 2ª vez:

git pull

git add .
git commit -m "Nome explicativo do Commit"
git push

Pull/Push DESCARTANDO alteração do Github que não está no repositório remoto

git pull -f origin main = git pull NomeDoRemote NomeDaBranch (trás da núvem destruindo commits posteriores local)
git push -f = manda para núvem destruindo commits posteriores da núvem

Apagar commit mantendo as alterações

git reset --soft HEAD~1 = apaga commit sem retirar as alterações

Apagar commit apagando as alterações

git reset --hard códigoDoCommit = apaga o commit e também apaga as alterações a partir do codigo do commit informado

Clonar projetos

git clone https://github.com/teodorolucasac/Github-Nelio-Alves.git = cria um clone do repositório remoto para pasta atual local

Apontar um repositorio remoto para outro repositorio remoto

git remote set-url origin URLdoRepositorioRemoto
git push -u origin main

Abrir VS Code

code . = abre vscode na pasta atual

GERAL

dir = lista todas as pastas do local
ls = mostra os arquivos da pasta
ls -all = mostra todos arquivos da pasta, inclusive ocultos
clear = limpa a tela do git
git status = status do versionamento da pasta atual
git log = mostra histórico de versões
git log --oneline = histórico de versões resumido

git config --global core.autocrlf false = impede o git de modificar as quebras de linhas

(aperta ESC) :q = Sair do editor de texto Vim

git checkout código-do-commit = volta para um commit anterior do código informado
git checkout main = volta para o ultimo commit da branch main (ou branch informada)
git checkout HEAD~2 = volta para 2 commits anteriores do main

git reset = retira as modificações do Stage
git restore --staged nomeDoArquivo.txt ou c:/caminhoda/pasta01 = retira um arquivo ou pasta específica do stage
git clean -df = limpa alguma modificação na branch 01/02
git checkout -- . = limpa alguma modificação na branch 02/02

Editor VIM

i = habilitar o modo de edição

<ESC>
:wq
<ENTER> = Sair do VIM, salvando as alterações

<ESC>
:q!
<ENTER> Sair do VIM, descartando as alterações

CTRL + q = Apenas sai do Vim

CTRL + c = Sai de algum comandos

  1. git clone https://github.con/uri... | git pull (origin master?) | rm -rf NomePasta (remove pasta e git) -> git clone...
  2. Criar nova branch -> git checkout -b feature/nome-da-branch
  3. Fazer as alterações do código
  4. git status,
