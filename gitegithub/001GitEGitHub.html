Git

Git é um software de de controle de versão (VCS)

Quando falamos em versão falamos de versionamento. Git é uma ferramenta de versionamento de código. Ou seja, o Git gerencia e organiza todas as versões de um projeto.

A grande importância dele se dá porque o programador lida sempre com muitas versões de algo. Exemplo:

Versionamento Centralizado/Linear

É o primeiro modelo de software de versionamento criado.

Basicamente esse VCS faz um commit em um repositório central, mas eles tiam uma grande limitação, para dar um commit era necessário ter conexão ativa com esse servidor constantemente, não era possível ter monitoramento no código sem uma conexão constante.

Uma observação que o servidor não cria um monte de zip para separar as versões, pois o software é capaz de perceber uma versão anterior e as diferenças e armazenar isso.

Versionamento Distribuído

Quando uma equipe está trabalhando no projeto, ela não commitam no repositório central, eles commitam no repositório local. Existe um software de versionamento instalado no seu computador. Agora eu não preciso do poder de processamento de um servidor e nem sequer estar conectado a ele.

Então como trabalhar com equipe remota?

É ai que nasce o conceito de repositório remoto (diferente de repositório central). Sempre que você chegar em um ponto do projeto que deseja registrar/salvar, basta dar um push (diferente de commit), e enviar ao repositório remoto.

Principais Vantagens

  • Controle de histórico
  • Trabalho em equipe
  • Remificação do projeto
  • Segurança
  • Organização

GitHub

Lembra que na explicação do Git ensinamos que você salvaria os projetos no repositório local, e quando o projeto chegasse em uma determinada versão, você daria um push pro repositório remeto? (diferente do repositório central).

Pois bem, o Git mais precisamente é o nosso repositório local, e o nosso repositório remoto é o GitHub.

O que o GitHub faz?

  • Repositórios ilimitados
  • Hospedagem de código-fonte
  • Características de rede social
  • GitHub Pages integrado (hospedagem do site)
  • Colaboração
  • Forks ()

Então o GitHub além de gerenciar seu código com maestrie permitindo,visualizar, colaborar, testar, sugerir alterações no código de qualquer um que disponilizar o código publicamente. Também faz muitas outras coisas.

Obs: existem outros repositórios remotos, mas um dos mais usados é o GitHub e é dele que vamos falar.
