Estudo sobre Semântica

No HTML podemos criar uma página de muitas formas diferentes como, só usando divs ou até mesmo usando tabelas. No entando isso dificulta a leitura, manutenção do código, edição, interpretação e ainda facilita a existencia de bugs

Com isso surgiu a marcação semântica no HTML5, que basicamente são Tags que não são usadas para gerar um conteúdo visível inicialmente, mas sim organizar e informar "dar nome aos bois" sobre o que representa aquela item, bloco, etc...

Exemplo, no lugar de colocar uma tag <div></div> para criar o cabeçalho, nos temos a tag <header></header> que faria praticamente a mesma coisa, mas informa e organiza, deixando explicito para o programador e a máquina interpretadora que aquilo é um cabeçalho.
