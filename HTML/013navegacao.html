Navegação

No HTML a navegação (utilizadas para criar menus) é representada pela tag semântica <nav>. Seguido de tags <a> que serão os links. Desta forma já é possível criar nossos Menus.

No entanto, normalmente usaremos também as tags <ul> seguido de <li> e depois <a> para que nossos menus tenha uma estrutura de listas não ordenadas.

Leitor de texto

Quando usamos listas o leitor de texto vai anúnciar o menu com mais detalhes e portanto mais acessibilidade, como quantidade de itens.

Já houve diversas discuções para definir se a navegação deve ser utilizada com lista ou sem lista.
Na conclusão foi constatado que com lista é uma melhor forma de utilizar a navegação, mas vejamos os pros e contras:

Navegação “Contra” em listas

- Pelo menos um usuário de leitor de tela prefere a navegação sem listas, que foi a origem do artigo original.
- Marcação mais simples. nav > aé mais fácil/menos que nav > ul > li > a.
- Divs e spans podem ser tão bons, especialmente com funções ARIA

Navegação “Para” em listas

- Anunciar o número de itens na lista pode ser útil
- Benefício para estruturar em navegadores não CSS (captura de tela do Lynx )
- Padrão de longa data que não provou ser um grande problema
- As listas são um “gancho” para leitores de tela (por exemplo, pressione L para listas) e exibem bem a hierarquia
- Segurança: nada pode estar em listas, mas itens de lista, não para navegação
