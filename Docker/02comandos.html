Comandos Docker:

-

Dockerfile

FROM - (imagem) especifica a imagem que o docker vai carregar, sistema operacional e plataforma (exemplo: node:12-alpine)
WORKDIR - Setta o diretório onde o docker irá trabalhar
COPY - Copia arquivos para a imagem
ADD - Adiciona arquivos para a imagem (web site ou zip)
ENV - Configurações do ambiente
RUN - Instala as dependencias e executa a aplicação enquanto está construindo a imagem
CMD - Executa algum arquivo depois que a imagem já está rodando
EXPOSE - Coloca a porta que será usada
USER - Se necessário, informa o usuário que usará a aplicação
ENTRYPOINT -

Geral Docker

docker login - vai fazer login no docker pelo terminal (se já estiver logado no docker desktop não pedirá credenciais)
[comando] --help - vai listar as opções do comando
docker cp [arquivo origem] [arquivo destino] - transfere do container pro local e do local pro container
-

Image Docker

docker images - mostra todas as imagens do seu docker
docker pull [nome da imagem] - baixa uma imagem do DockerHub
docker build -t [nome da imagem] . - cria uma imagem considerando o Dockerfile dentro do diretório do terminal
docker build -t [nome da image]:[numero da versão] . - cria uma imagem com uma versão específica
docker image tag [nome da imagem:numero da versão] [nome da imagem:numero da versão] - troca a tag/versão do da imagem
docker push [nome do repositório]/[nome da imagem:nome da versão] - faz um pull para DockerHub
-

Container Docker

docker ps - mostra todos containers rodando
docker ps -a - mostra todos os caintainers rodando e parados
docker run [nome da imagem] - roda uma imagem da máquina ou do DockerHub em um container
docker run -it [nome da imagem] - roda uma imagem da máquina ou do DockerHub no modo Interactive
docker run -d [nome da imagem:versão] - sobe um container em background, liberando o terminal
docker run -d --name [nome do container] [nome da imagem:versão] - sobe container com nome específico
docker run -d -p 80:3000 --name [container name] [nome da imagem:versão] - inicia um container com porta de entrada do host:container
docker logs [container id] - mostra os logs do container
docker exec [container name] ls - executa comandos dentro do container sem acessar o container, no caso mostrou os arquivos
docker rm [container name ou id] - remove um container
docker rm -f [container name ou id] - remove um container de modo forçado
docker run -d -p [3000:3000] --name [kiwi] -v [app-dados:/app/dados] [app:v2] - cria um container e insere um volume (backup) informando [nome do volume]:/[diretorio em que ficará no container] e tudo que ele armazenar neste diretório do container, fará um backup na máquina local
-

Volume Docker

docker volume create [nome do volume] - cria um volume
docker volume inspect [nome do volume] - verifica os dados do volume, inclusive seu diretório local
-

Docker-Compose

docker-compose up - executa seu arquivo docker-compose.yml e cria uma imagem apenas se ela não existir
docker-compose up --build - executa seu arquivo docker-compose.yml e obrigatóriamente cria uma imagem
docker-compose -d - executa seu arquivo docker-compose.yml em background
docker-compose ps - lista os container em execução
docker-compose down - stopa o container
docker exec -it -u root [container id] sh - entra no container informado em modo interactive
docker-compose logs - mostra todos os logs
docker-compose logs --help - mostra as opções de logs

YAML

Data Serialization, muito utilizada para escrever configurações, através de instruções no qual um aplicativo lerá de cima para baixo.

Comandos Linux:

Principais Linux

ls - lista todos os arquivos e diretórios do local
ls -1 - lista todos os arquivos e diretórios do local um em baixo do outro
ls -l - lista todos os arquivos e diretórios do local mais detalhadamente
whoami - mostra qual usuário você está usando
echo - imprime uma mensagem na tela
echo $0 - mostra o diretório ou pasta que você está
pwd - print working diretory, mostra em qual diretório você está
history - mostra uma enumerado de últimos comandos utilizados no terminal
cd /[nome do diretório] - (change diretory) move até o diretório descrito
cd .. - volta um diretório
cd [tab 2x] - lista as possíveis pastas para navegar
cd [nome do diretório parcialmente] + [tab] - auto completa o nome do diretório para navegar
ls /[diretório]/[diretório] - lista os arquivos e diretórios de uma pasta que você não está
cd ~ - entra dentro do diretório home do usuário root
cat [nome do arquivo] - visualiza / imprime o conteúdo do arquivo
more [nome do arquivo] - visualiza / imprime o conteúdo do arquivo por partes
cat [nome do arquivo] > [nome do arquivo] - cria um novo arquivo com conteúdo do antigo, podendo passar mais de um arquivo para "cat"
echo [sua mensagem] > [nome do arquivo] - cria um arquivo com a mensagem do echo
grep [nome da pesquisa] [nome do arquivo] - (global regular expression print) pesquisa uma palavra dentro de um arquivo
grep -i -r [pesquisa] . - pesquisa em todos os arquivos do diretório (-i ignora CaseSensitive) (-r procura dentro de subdirectory)
find - mostra todos os diretórios, inclusive os ocultos
find -type f - mostra todos os arquivos (files)
find -type d - mostra todos os diretórios
find -type f -name "[nome]" - mostra o arquivos com aquele nome (é case sensitive)
mkdir [nome pasta] ; cd [nome pasta] ; echo [mensagem] - o ; faz executar vários comandos de uma vez sem parar nos erros
mkdir [nome pasta] && cd [nome pasta] && echo [mensagem] - o && faz executar vários comandos de uma vez parando nos erros
ps - mostra os processos rodando no linux
kill [process id] - mata os processos que estão rodando no linux
useradd - adiciona um usuário
usermod - modifica um usuário
userdel - deleta um usuário
useradd -m lucas - criou um usuario com diretório home chamado de lucas
docker exec -it -u root 0ccc9387e271 bash - acessa meu container linux pelo modo interactive com usuário root através do bash
groupadd [nome do grupo] - cria um grupo de usuário
usermod -G docker lucas - coloca o usuário "lucas" dentro do grupo "docker"

Criar e Remover diretórios

mkdir [nome do diretório] - cria um diretório dentro do diretório em que você está
mv [nome do diretório] [novo nome] - renomeia o nome do diretório
touch [nome do arquivo].[tipo do arquivo] - (exemplo touch texto.txt) cria um arquivo
rm [nome do arquivo] - remove um arquivo
rm -r [nome do diretório] - remove diretório
-

Instalar e Remover Packages (software)

apt list - mostra lista de softwares instalados no seu linux
apt update - acessa o ubunto e faz download dos packages mais utilizados e mais atualizados
apt install [nome do software] - instala um package (software)
apt remove [nome do software] - desinstala um package (software)
